# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 1.0.x   | :white_check_mark: |

## Reporting a Vulnerability

We take the security of PhishGuard AI seriously. If you discover a security vulnerability, please follow these steps:

1. **Do Not** open a public issue
2. Email details to: subhajitroy857@gmail.com
3. Include:
   - Description of the vulnerability
   - Steps to reproduce
   - Potential impact
   - Suggested fix (if any)

## Response Timeline

- **Initial Response**: Within 48 hours
- **Status Update**: Within 7 days
- **Fix Timeline**: Depends on severity

## Security Best Practices

When using PhishGuard AI:

- Keep dependencies updated
- Use HTTPS in production
- Implement rate limiting on API endpoints
- Sanitize all user inputs
- Never commit sensitive data (API keys, credentials)

Thank you for helping keep PhishGuard AI secure!
